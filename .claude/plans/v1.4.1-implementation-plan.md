- Find the corresponding draft session by set code and time window
- Insert into `draft_match_results` table when match is saved
- Files: `internal/mtga/log/processor.go`, `internal/mtga/match/service.go`

**Tests Required:**
- Unit: Match detection identifies draft matches correctly

- E2E: Complete draft, play matches, verify linkage

**Acceptance Criteria:**
- [ ] Draft matches are detected during log processing
- [ ] Matches are linked to draft sessions in `draft_match_results`
- [ ] Community Comparison works without fallback query

#### #909: Standardize JSON Field Naming Convention
**Problem:**
REST API handlers use snake_case (`deck_id`, `max_results`) while frontend types use camelCase (`deckID`, `maxResults`). This causes 400 errors when frontend sends requests.

**Files to update:**
- `internal/api/handlers/draft.go`
- `internal/api/handlers/match.go`
- `internal/api/handlers/card.go`
- `internal/api/handlers/deck.go`
- `internal/api/handlers/export.go`

**Acceptance Criteria:**
- [ ] Audit all request/response structs in handlers
- [ ] Update JSON tags to use camelCase (matching frontend)
- [ ] Add integration tests for affected endpoints
- [ ] Document convention in CONTRIBUTING.md
