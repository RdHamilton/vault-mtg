# CloudFormation Templates — vaultmtg.app CDN\n\n## Overview\n\nThis directory contains CloudFormation templates for deploying the VaultMTG application infrastructure on AWS.\n\n### `vaultmtg-app-cdn.yaml`\n\nA single CloudFormation stack covering both `vaultmtg.app` (marketing site) and `app.vaultmtg.app` (React SPA).\n\n**Resources:**\n- 2 S3 buckets (private, versioned, encrypted)\n  - `vaultmtg-app-marketing` — marketing site static assets\n  - `vaultmtg-app-spa` — React SPA static assets\n- 1 ACM certificate covering vaultmtg.app, www.vaultmtg.app, app.vaultmtg.app (DNS-validated)\n- 2 CloudFront distributions with Origin Access Control (OAC)\n  - Marketing site: aliases vaultmtg.app, www.vaultmtg.app\n  - React SPA: alias app.vaultmtg.app\n- 2 S3 bucket policies (grant CloudFront OAC read-only access)\n\n**Cache & Routing:**\n- Both distributions use AWS-managed `CachingOptimized` policy\n- Custom error handling: 404 errors redirect to `/index.html` with 200 response (SPA fallback)\n- HTTP automatically redirected to HTTPS\n- PriceClass_100 (US/EU only)\n\n**Template Parameters:**\n| Parameter | Type | Default | Description |\n| --- | --- | --- | --- |\n| `Environment` | String | `production` | Deployment environment tag (production, staging) |\n\n## Deployment\n\n### Prerequisites\n- AWS CLI configured with `--profile personal` and credentials in `~/.aws/credentials`\n- AWS account with permissions to create S3, ACM, CloudFront resources\n- Route 53 hosted zone for `vaultmtg.app` (already exists)\n\n### Step 1: Deploy the Stack\n\n```bash\ncd infrastructure/cloudformation\n\naws cloudformation deploy \\\n  --template-file vaultmtg-app-cdn.yaml \\\n  --stack-name vaultmtg-app-cdn \\\n  --parameter-overrides Environment=production \\\n  --profile personal \\\n  --region us-east-1 \\\n  --no-fail-on-empty-changeset\n```\n\nThe stack creates all resources. **The ACM certificate will be in PENDING_VALIDATION state** — it won't validate until you add DNS CNAME records.\n\n### Step 2: Validate the ACM Certificate via DNS\n\n1. Go to [AWS Certificate Manager Console](https://console.aws.amazon.com/acm/) (us-east-1)\n2. Find the certificate with description \"S3 buckets, ACM certificate, CloudFront OAC distributions...\"\n3. Click the certificate ID to open details\n4. In the **Domains** section, you'll see CNAME records needed for validation:\n   - One CNAME for `_xxxxx.vaultmtg.app.`\n   - One CNAME for `_xxxxx.www.vaultmtg.app.` (if SANs enabled)\n   - One CNAME for `_xxxxx.app.vaultmtg.app.`\n5. Go to [Route 53 Hosted Zones Console](https://console.aws.amazon.com/route53/)\n6. Open the hosted zone for `vaultmtg.app`\n7. Create CNAME records matching the ACM certificate's validation requirements\n8. DNS validation completes automatically (typically 5–30 minutes)\n\n### Step 3: Point DNS to CloudFront\n\nOnce the ACM certificate is validated, create Route 53 A records (ALIAS type) pointing to CloudFront:\n\n1. Go to Route 53 > Hosted Zones > vaultmtg.app\n2. Create ALIAS records:\n   - **Record name:** `vaultmtg.app`\n   - **Record type:** A (ALIAS to CloudFront distribution)\n   - **Alias target:** CloudFront distribution domain for marketing site (output: `MarketingDistributionDomain`)\n   - **Routing policy:** Simple\n3. Create another ALIAS record:\n   - **Record name:** `www.vaultmtg.app`\n   - **Record type:** A (ALIAS to CloudFront distribution)\n   - **Alias target:** CloudFront distribution domain for marketing site\n   - **Routing policy:** Simple\n4. Create another ALIAS record:\n   - **Record name:** `app.vaultmtg.app`\n   - **Record type:** A (ALIAS to CloudFront distribution)\n   - **Alias target:** CloudFront distribution domain for React SPA (output: `SPADistributionDomain`)\n   - **Routing policy:** Simple\n\n### Step 4: Retrieve Stack Outputs\n\nAfter the stack is created and ACM certificate is validated:\n\n```bash\naws cloudformation describe-stacks \\\n  --stack-name vaultmtg-app-cdn \\\n  --query 'Stacks[0].Outputs' \\\n  --profile personal \\\n  --region us-east-1 \\\n  --output table\n```\n\n**Key outputs:**\n- `MarketingBucketName` — use for deploying marketing site assets\n- `SPABucketName` — use for deploying React SPA build artifacts\n- `MarketingDistributionId` — use for CloudFront cache invalidation after deployments\n- `SPADistributionId` — use for CloudFront cache invalidation after deployments\n\n## Deployment Pipeline Integration\n\n### Build & Deploy Marketing Site\n\n```bash\nnpm run build  # in marketing site project root\naws s3 sync dist/ s3://vaultmtg-app-marketing/ --delete --profile personal\n\n# Invalidate CloudFront cache\naws cloudfront create-invalidation \\\n  --distribution-id <MarketingDistributionId> \\\n  --paths '/*' \\\n  --profile personal\n```\n\n### Build & Deploy React SPA\n\n```bash\nnpm run build  # in frontend/ (React Vite SPA)\naws s3 sync dist/ s3://vaultmtg-app-spa/ --delete --profile personal\n\n# Invalidate CloudFront cache\naws cloudfront create-invalidation \\\n  --distribution-id <SPADistributionId> \\\n  --paths '/*' \\\n  --profile personal\n```\n\n## Troubleshooting\n\n### CloudFormation Deployment Fails\n\n- Check IAM permissions: ensure the `personal` profile has `s3:*`, `cloudfront:*`, `acm:*` permissions\n- Check region: template must be deployed in `us-east-1` (ACM certificate requirement for CloudFront)\n\n### ACM Certificate Validation Stuck\n\n- Verify CNAME records were created in Route 53\n- Check CNAME values match exactly (including trailing dots if present)\n- Wait 10–30 minutes for DNS propagation\n- In Certificate Manager console, click \"Resend email\" or refresh to check validation status\n\n### CloudFront Returns 403 Forbidden\n\n- Ensure S3 bucket policy grants CloudFront OAC read access (template includes this)\n- Check bucket is not accidentally public (template blocks public access)\n- CloudFront OAC must have the correct origin (check CloudFront distribution config)\n\n## Architecture Decision\n\nSee [ADR-008: Frontend Serving Model](../../docs/adr/ADR-008-frontend-serving-model.md) for the architecture rationale: S3+CloudFront as canonical, Vercel for preview-only, EC2 for API only.\n\n## Security Notes\n\n- **S3 buckets:** All public access blocked. Only CloudFront OAC can read.\n- **HTTPS:** All HTTP traffic redirected to HTTPS. TLS 1.2+.\n- **Cache:** CachingOptimized managed policy ensures optimal performance without exposing secrets.\n- **Encryption:** S3 buckets use AES256 server-side encryption (envelope encryption handled by CloudFront).\n"
