# MTGA Player.log Event Taxonomy — build 2026.59.20

Source capture: `Player_capture_premier_20260531T072112Z.log` (57,339 lines, Detailed Logs ON, single cumulative session: QuickDraft (BotDraft), a full Premier Draft, 5 matches, plus quest/inventory/rank/periodic-rewards surfaces). Raw log is PII and is never committed.

Generated by the `--catalog` mode of `tools/fixture-extractor/extract.py` (ticket #262). The full machine catalog + one sanitised sample per distinct event type are produced by that tool; this document is the human taxonomy + daemon gap map that feeds daemon-roadmap scoping.

## Summary

**101 distinct event types across 6 structural axes** (the catalog also surfaces a 7th `prefix-message` axis for bare-prefix notifications such as `Draft.Notify`, giving 106 rows total). MTGA's log is NOT a flat stream of business-event JSON — there are several structural axes and the daemon only consumes a handful of them.

| axis | distinct | what it is | daemon coverage |
|---|---|---|---|
| `api-request` (`==>`) | 19 | client→server API calls; body often a stringified `request` envelope | mostly ignored (deck/draft-pick partial) |
| `api-response` (`<==`) | 19 | server→client responses; body often a stringified `Payload` envelope | mostly ignored |
| `prefix-message` | 5 | bare-prefix notifications (`Draft.Notify`, `Client.SceneChange`, …) | `Draft.Notify` = Premier pack (parser drifted) |
| `gre-message` (`GREMessageType_*`) | 22 | per-turn game-rules-engine moves | ignored (only `match.game_ended` telemetry partially consumed) |
| `client-message` (`ClientMessageType_*`) | 20 | client→GRE move submissions | ignored |
| `match-room-state` (`MatchGameRoomStateType_*`) | 2 | match lifecycle | `MatchCompleted` parsed & emitted; `Playing` ignored |
| `json-key` | 19 | single-line top-level business JSON (envelope-unwrapped) | auth / quests / inventory parsed & emitted |

## Axis: per-turn GRE + client moves (`GREMessageType_*` / `ClientMessageType_*`)

The entire 22 GRE + 20 client message types are **ignored** by the daemon today (only `match.game_ended` telemetry is partially derived from the GRE session buffer). These are the per-turn move universe: `GameStateMessage` (416 + 994 UI), `ActionsAvailableReq` (119), `SelectTargetsReq` (43), `DeclareAttackersReq` (20), `MulliganReq` (6), `DieRollResultsResp` (5), `ChooseStartingPlayerReq` (3), etc.; and the client-side submissions `PerformActionResp` (119), `SubmitTargetsReq` (21), `MulliganResp` (13), `SubmitAttackersReq` (12), `ConcedeReq` (2), etc. Scoping per-turn capture is a large future ticket; raw shapes are sampled in the catalog `samples/gre-message__*.json` + `samples/client-message__*.json`.

## Known gaps called out for roadmap scoping

1. **match-start** — `MatchGameRoomStateType_Playing` + `EventJoin` + `EventEnterPairing` (daemon emits nothing at match start; only `match.completed` at the end).
2. **per-turn GRE moves** — all 22 `GREMessageType_*` + 20 `ClientMessageType_*` ignored.
3. **daily-win / weekly-win / weekly rewards** — `PeriodicRewardsGetStatus` response + `_dailyRewardSequenceId`/`_weeklyRewardSequenceId` + reset timestamps.
4. **rank** — `RankGetCombinedRankInfo` / `RankGetSeasonAndRankDetails` + `constructedSeasonOrdinal/Level/Step`, `limitedRankInfo`.
5. **Premier draft classifier drift** — Premier pack (`Draft.Notify`) and pick (`EventPlayerDraftMakePick`) are no longer top-level `draftPack`/`pickedCards`, so the daemon classifies 0 Premier draft events. Sibling of #336.
6. **BotDraft (QuickDraft) support** — `BotDraftDraftPick`/`BotDraftDraftStatus` (+ `DraftPack` capitalised envelope). Tracked separately by **#337**; not promoted in #262.

Note: vault % is already captured via `inventory.updated.TotalVaultProgress` (not a gap).
