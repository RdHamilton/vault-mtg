# Archive

Files here are preserved for institutional history. **Do not reference them from agent instructions or active PRDs.**

## Subdirectories

### `desktop-era/`
Content predates the cloud migration (pre-2026). The product was originally a desktop Electron application with local storage and a built-in deck builder. These docs describe systems that no longer exist (local SQLite, in-app collection editor, MVP UX explorations). Kept for product history and the rare case an old design choice needs to be re-examined.

### `completed-plans/`
Finished sprint plans, superseded specs, one-off engineering drafts that were rolled into other docs, and gap analyses that have been resolved. Anything in here is "done" — neither aspirational nor active.

## Policy

- New work should never cite an archived doc as authoritative.
- If an archived doc contains a still-relevant section (e.g., a useful architectural rationale), extract that section into a current doc under `docs/architecture/`, `docs/engineering/`, or the appropriate live tree, and leave the archive copy untouched.
- Do not delete archived files; the goal is institutional memory, not minimalism.
